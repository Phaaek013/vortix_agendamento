ClinicOps é um ecossistema SaaS *multi-tenant* focado em revolucionar clínicas de saúde. Uma plataforma desenhada tela-a-tela para entregar experiência "nível A" tanto para pacientes quanto para administradores e profissionais da saúde. O projeto prioriza consistência de UI, interatividade instantânea e tema noturno impecável.

## 🚀 Tecnologias e Stack

A ClinicOps foi construída em cima das ferramentas mais modernas para assegurar perfomance, escalabilidade e facilidade de manutenção.

*   **⚡ Vite**: Build tool ultrarrápida que impulsiona o desenvolvimento.
*   **⚛️ React 19**: A base do projeto, utilizando a versão mais recente e veloz.
*   **🔵 TypeScript**: Tipagem estática em 100% da base de código para evitar surpresas amadoras e bugs em produção.
*   **💅 Tailwind CSS 4**: O motor visual do projeto. Responsável pelo Dark Mode absoluto, responsividade cirúrgica e paletas refinadas sem perder consistência (fim do "Frankenstein UI").
*   **🧭 React Router (v7)**: Gestão de navegação fluída estilo SPA, com manipulação avançada de estados por rotas.

## 🎯 Highlights do Empreendimento

- **Design Premium**: Esqueça telas básicas e genéricas! Desenhamos uma interface que causa "Uau" instantâneo em qualquer clínica.
- **Dark Mode Perfeito**: Dezenas de microajustes precisos para garantir que tabelas, cards KPIs numéricos (no Dashboard) e filtros drop-downs fiquem nítidos e bonitos no tema escuro.
- **Ecossistema Limpo**: Um repositório "lisão", sem logs polutivos, livre de quebras de compilação (*passed 100% on ESLint/Vite*) ou rastro de amadorismos.
- **Arquitetura Multi-Tenant**: Estruturas de pastas, referências cruzadas e rotas desacopladas de *hardcodes* comissionando qualquer nova marca ou clínica facilmente.
- **Fila de TV & Atendimento**: Telas autônomas orientadas ao dia-a-dia logístico da recepção de uma grande clínica.

## 📦 Como rodar localmente

Clone o projeto e rode com Node 20+:

```bash
# 1. Instalar as dependências
npm install

# 2. Rodar o servidor de desenvolvimento ultra-rápido (Vite)
npm run dev

# 3. Rodar o linter de verificação final
npm run lint

# 4. Gerar a build de produção
npm run build
```
